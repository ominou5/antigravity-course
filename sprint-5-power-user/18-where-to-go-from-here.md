## Extending Your Habit Tracker

The app is live — but it's just a starting point. Here are ideas to keep building:

**Features to add:**
```
Add a feature that automatically resets all habits to "unchecked" 
at midnight each day, so users start fresh every morning.
```

```
Add a habit history page that shows which habits were completed 
on which days over the past 30 days. Display it as a simple calendar 
or grid view.
```

```
Add a "progress" section to the dashboard showing:
- How many habits are tracked
- Current longest streak
- Habits completed today vs. total
```

**Make it shareable:**
```
Add a feature where users can share a public "streak page" 
showing their habit streaks — a simple read-only view at /[username]
```

**Build a mobile version:**
```
Convert my Habit Tracker into a Progressive Web App (PWA) 
so users can install it from their phone's browser like a real app.
```

## Going Deeper With Antigravity

As you get more comfortable, explore these advanced capabilities:

**Frameworks:**
When you're ready to graduate beyond vanilla JS, try Vite or Next.js:
```
I want to rebuild my Habit Tracker using Vite as a build tool 
to make it more scalable. Walk me through the migration.
```

**Custom agents:**
Antigravity can help you build and deploy custom AI agents — chatbots, assistants, automation tools that run independently:
```
I want to create a simple AI agent that sends me a daily 
SMS reminder about my habits. Help me plan this.
```

**API integrations:**
Connect your app to external APIs using the MCP ecosystem or by fetching data directly:
```
Connect my Habit Tracker to a weather API so it suggests 
habit adjustments based on today's weather 
(e.g., "It's raining — replace your outdoor run with a home workout").
```

Here are prompts to bookmark for when you start your next build:

```

```

**Stuck on a bug:**
```
I have a bug I can't figure out. Here's what's happening: [describe].
Here's what I've tried: [describe]. Here's the relevant code: [paste].
Walk me through debugging this step by step.
```

**Learning a new technology:**
```
I want to learn [technology — e.g. React, TypeScript, Node.js].
Teach me by building something practical with it.
Start from absolute basics and explain everything as we go.
Don't move on until I confirm I understand each step.
```

**Getting unstuck on design:**
```
My app works but it looks dated and feels cheap.
Review the current design and propose 5 specific visual improvements, 
then apply whichever ones I approve.
```

## The Mindset That Makes It Work

A few things that separate builders who ship from those who stall:

**Commit to small steps.** Break everything into the smallest possible task. Done beats perfect.

**Use Antigravity as a tutor, not just a builder.** Ask "why," ask for explanations, ask it to teach you. The more you understand, the better your prompts get.

**Don't fear mistakes.** Git means nothing is permanent. Break things. Fix them. Learn.

**Ship early, improve continuously.** Your first deployed app doesn't have to be perfect. It just has to be real.
